# 100. detect circle in linked list

A [Singly Linked List](https://en.wikipedia.org/wiki/Linked_list#Singly_linked_list) is a bunch of nodes linked in one direction.

```js
class Node {
  val: any
  next: Node
  constructor(val: any, next:Node) {
    this.val = val
    this.next = next
  }
}
const node2 = new Node(2)
const node1 = new Node(1, node2) // connect 1 -> 2
```

A Node might link to a node before it, thus creating a circle.

Can you write a function to detect it?

What is the space cost for your approach? Can you solve it **without extra space**?

```js
function hasCircle(head) {
  // ğŸ’¡ Check if the list is empty; if yes, no cycle can exist
  if (head === null) return false;
  
  // ğŸ’¡1 Initialize slow pointer at the head node
  let slow = head;
  // ğŸ’¡1 Initialize fast pointer at the head node
  let fast = head;

  while (fast !== null && fast.next !== null) {
    // ğŸ’¡R1: slow moves to 2
    // ğŸ’¡R2: slow moves to 3
    // ğŸ’¡R3: slow moves to 4
    // ğŸ’¡R4: slow moves to 5
    // ğŸ’¡R5: slow moves to 1 (cycle detected if fast also at 1)
    slow = slow.next;         // Move slow pointer one step
    
    // ğŸ’¡R1: fast moves to 3
    // ğŸ’¡R2: fast moves to 5
    // ğŸ’¡R3: fast moves to 1
    // ğŸ’¡R4: fast moves to 3 (cycle confirmed if slow reaches 3 in next step)
    fast = fast.next.next;    // Move fast pointer two steps
	
    // ğŸ’¡R1: slow at 2, fast at 3 (no match)
    // ğŸ’¡R2: slow at 3, fast at 5 (no match)
    // ğŸ’¡R3: slow at 4, fast at 1 (no match)
    // ğŸ’¡R4: slow at 5, fast at 3 (no match)
    // ğŸ’¡R5: slow at 1, fast at 1 (match - cycle detected)
    if (slow === fast) {
      return true; // Cycle detected
    }
  }

  // ğŸ’¡ If we reach here, fast has encountered the end of the list, meaning no cycle exists
  return false; // No cycle detected
}
```
